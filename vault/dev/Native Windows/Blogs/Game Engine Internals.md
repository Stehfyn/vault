# Game Engine Internals

Fabien Sanglard's Game Engine Black Books are here because they explain old PC engines at the level Windows programmers still need: memory layout, fixed-point math, renderer constraints, file formats, timing, and the hardware/API assumptions that shaped the code. They are more useful than generic nostalgia because the books connect architecture decisions to concrete machine limits.

Use this as background for graphics, input, and timing entries. Wolfenstein, Doom, and Quake-era engines show why game loops, frame pacing, software blitters, and platform abstraction evolved the way they did before Direct3D-era assumptions became standard.

Code contribution: connect the books to fixed-step loops, frame timing, software blitters, asset pack readers, `BSP` traversal, and platform abstraction boundaries used before D3D-era renderer structure.
