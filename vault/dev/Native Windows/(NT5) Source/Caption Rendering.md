# Caption Rendering (caption.c)

`caption.c` is the NT5 win32k side of non-client painting: it computes caption rectangles, frame metrics, active/inactive states, system-menu glyphs, and caption-button hit targets before USER32 hands control back to application code. The useful lesson is that title bars are not a separate widget. They are negotiated through `WM_NCCALCSIZE`, `WM_NCPAINT`, `WM_NCHITTEST`, style bits, system metrics, and default-window-proc behavior.

Modern custom-frame code still pays this debt. Removing or replacing the non-client area means taking responsibility for hit-testing, snap/resize affordances, accessibility expectations, DWM composition, and system-command routing. `caption.c` is valuable because it shows the old canonical implementation before DWM moved more drawing policy out of classic win32k.

## Connections
- `System Command Processing` handles the commands generated by caption buttons and system-menu actions.
- `MinMax Size Computation` and `Window Manager Internals` define the resize/move constraints around the same non-client frame.
