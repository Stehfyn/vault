# .NET 3.5 in Windows Sandbox 24H2 Guide

This is a narrow procedural note for one failure mode: enabling NetFx3 inside a Windows Sandbox instance where the normal DISM path cannot reach Windows Update and the sandbox image does not already contain the Feature-on-Demand payload. The useful part is not ".NET 3.5 install" in the abstract; it is the sandbox-specific sequence of clearing WU policy friction, repairing the component store when necessary, and pointing DISM at an offline `sources\sxs` payload before the ephemeral VM is discarded.

Keep the entry with the Windows installer and app-compat material, not with modern .NET. It exists because old WinForms, MFC-hosted CLR add-ins, COM-visible .NET 2.0/3.5 components, and vendor tools still show up when testing native Windows software in a clean throwaway desktop.
